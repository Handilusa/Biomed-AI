# Evidence Bundle Documentation

This document describes the structured logging format and demo run procedure for verifying Edge MedTech Copilot's performance claims.

---

## Demo Run

### Running the Demo

```bash
npm run demo:log
```

This executes a fixed battery of 5 queries and generates structured logs in the `logs/` directory.

### What the Demo Does

1. **Loads models** - MedPsy-4B Q4_K_M + embedding model via QVAC SDK
2. **Classifies intent** - Deterministic keyword router assigns each query
3. **Retrieves context** - RAG search for technical queries
4. **Generates response** - Streaming completion via QVAC SDK
5. **Logs everything** - JSONL + CSV with all performance metrics

### Demo Queries

| ID | Type | Query |
|----|------|-------|
| tech-01 | Technical | SpO2 readings on our patient monitor are showing intermittent "probe off" errors even though the sensor is properly connected to the patient. What should I check? |
| tech-02 | Technical | Our infusion pump is showing an "occlusion" alarm but the IV line appears completely clear with no visible kinks. How do I troubleshoot this? |
| tech-03 | Technical | I am getting an error code E-22 on the patient monitor. What does this mean and how do I fix it? |
| tech-04 | Technical | The monitor turns on but powers off randomly after 30 seconds to 2 minutes. The battery shows 80% charge. |
| tech-05 | Technical | The SpO2 module calibration failed verification during preventive maintenance. The readings are out of tolerance by 4%. |
| tech-06 | Technical | The alarm limits reset to factory defaults every time the device is power cycled. Is this normal? |
| tech-07-es | Technical (ES) | La bomba marca oclusión pero la línea está libre y los settings de presión de oclusión están configurados al mínimo. |
| med-01 | Clinical (Safety) | What are the clinical risks if a mechanical ventilator delivers incorrect tidal volumes to a patient? Why is this dangerous? |

## Log Format

### JSONL Format (`demo_run_*.jsonl`)

Each line is a JSON object with the following fields:

| Field | Type | Description |
|-------|------|-------------|
| `timestamp` | string | ISO 8601 timestamp of the request |
| `request_id` | string | UUID identifying the request |
| `agent` | string | Agent that handled the query: `technical`, `medical`, or `router` |
| `model` | string | Exact model filename used (e.g., `medpsy-4b-q4_k_m-imat.gguf`) |
| `query` | string | The user's original query text |
| `intent` | string | Classified intent: `technical_device_issue`, `medical_educational_context`, or `other` |
| `prompt_tokens` | number | Number of tokens in the prompt sent to the model |
| `completion_tokens` | number | Number of tokens generated by the model |
| `ttft_ms` | number | Time to first token in milliseconds |
| `tokens_per_second` | number | Generation speed (tokens/second) |
| `total_time_ms` | number | Total wall-clock time for the completion |
| `rag_chunks_used` | number | Number of RAG document chunks included in context |
| `memory_mb` | number? | Estimated memory usage in MB (if available) |
| `has_disclaimers` | boolean | Whether medical disclaimers were included |

### Example Entry

```json
{
  "timestamp": "2026-06-15T10:30:00.000Z",
  "request_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "agent": "technical",
  "model": "medpsy-4b-q4_k_m-imat.gguf",
  "query": "The SpO2 readings on our patient monitor are showing intermittent 'probe off' errors...",
  "intent": "technical_device_issue",
  "prompt_tokens": 1250,
  "completion_tokens": 350,
  "ttft_ms": 420,
  "tokens_per_second": 28.5,
  "total_time_ms": 12800,
  "rag_chunks_used": 3,
  "has_disclaimers": false
}
```

### CSV Format (`demo_run_*.csv`)

Same fields as JSONL, exported as comma-separated values for spreadsheet analysis.

## Verification Checklist

For hackathon judges to verify coherence between code, logs, video, and hardware:

### 1. Code ↔ Configuration

- [ ] Model filename in logs matches `LLM_MODEL_FILE` in `.env` / `.env.example`
- [ ] QVAC config in `qvac.config.json` lists the correct plugins
- [ ] No cloud AI imports in `package.json` (no OpenAI, Anthropic, Gemini packages)

### 2. Logs ↔ Video

- [ ] Demo run query text matches what's shown in the video
- [ ] TTFT and TPS values in logs are consistent with video timestamps
- [ ] Agent routing (technical vs. medical) matches the visual badges in the UI

### 3. Hardware ↔ Performance

- [ ] Declared hardware matches system screenshots
- [ ] TTFT values are consistent with the declared hardware (RTX 5060 + 32GB RAM)
- [ ] No evidence of cloud API calls in the codebase or network traffic

### 4. Reproducibility

```bash
# Full reproduction from scratch
git clone https://github.com/Handilusa/Biomed-AI.git
cd Biomed-AI
npm install
npm run download:models
npm run ingest
npm run demo:log
# Inspect logs/demo_run_*.jsonl
```
